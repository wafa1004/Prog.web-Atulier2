Introduction Un hyperlien permet de lier un document HTML avec une autre ressource du Web: image, programme, document HTML (voir http://www.w3schools.com/), section dans un document HTML,
• Un hyperlien à 2 extrémités appelées ancres: la source: point de départ de l'hyperlien et la destination: point d'arrivée du lien.
• La destination de l'hyperlien est donnée par son URL
• Élément a (a pour ancre)

1. Adressage des ressources

1.1. Adresse de base

Chaque adresse URL a une adresse de base : c’est la partie de l’URL excluant ce qui suit le dernier « / ».
Par exemple :
L’adresse de base de http://www.uqtr.ca/~touref/INF1001/index.html est http://www.uqtr.ca/~touref/INF1001/ Habituellement le navigateur considère que l’adresse de base par défaut est l’adresse de base du document. La balise <base> dans l’entête du document permet donner une adresse de base par défaut différente de celle du document.

1.2. URL relatif

L'attribut href spécifie l'URL de la destination de l'hyperlien. Cet URL peut être
o Absolu: pour identifier une ressource indépendamment du contexte.
http://www.tuteurs.ens.fr/internet/web/html/bases.html
o Relatif: pour identifier une ressource dépendamment du contexte.
Ce contexte est donné par l'adresse de base.
Par exemple, si l'adresse de base est http://www.uqtr.ca/~touref/INF1001/documents/, alors l'URL relatif lien/lesport.jpg désigne l'URL absolu http://www.uqtr.ca/~touref/INF1001/documents/lien/lesport.jpg

2. Élément a

• Établit une relation entre une partie d'un document HTML et une autre ressource
• Permet de définir la source du lien et la destination du lien.
• L'attribut href donne l’URL de la destination du lien.
• Le contenu de la balise a est la source du lien
• L'activation de l'hyperlien (habituellement cliquer sur le lien) permet d'accéder à la ressource définie par la destination du lien. Tutoriel XHTML Source du lien:" Tutoriel XHTML " Destination de lien:" http://www.tuteurs.ens.fr/internet/web/html/bases.html "

2.1. Syntaxe

• L'élément a s'ajoute à l'ensemble de contenu Inline du module Text
• Le contenu de l'élément a doit être de niveau texte mais ne doit pas contenir d'élément a.

2.2. Définition d'une destination à l'intérieur du document

• Il est possible de faire un hyperlien vers une section du document.
• On utilise l'attribut id (pour identificateur) pour spécifier une section du document

OBJECTIFS SPÉCIFIQUES

• La valeur de l’attribut id a une syntaxe prédéfinie : Elle commence par
- une lettre,
- le caractère "_" ou
- le caractère ":" et
Les autres caractères sont
- des lettres,
- chiffres,
- le caractère "_",
- le caractère "-",
- le caractère ":" ou
- le caractère "."

• On peut par la suite utiliser cet identificateur comme destination d'un lien
• Il suffit d'utiliser le symbole # (dièse) devant le nom de la section comme valeur de l'attribut href. On parle alors d'identificateur de fragment. OBJECTIFS SPÉCIFIQUES
• L'identificateur de fragment peut être utilisé avec un adressage relatif ou absolu

3. Règles de style CSS


selon l'état de l'hyperlien En CSS on considère qu'un hyperlien peut être dans l'un des états suivants :
• link : un lien qui n’a pas encore été visité
• visited: un lien qui a été visité
• active: un lien pointé par l’usager.
Si on veut avoir des déclarations de style distinctes selon l’état d’un élément a on utilise ce qui est appelé en CSS des pseudo-classes. (On parle de pseudo-classe parce que ce qui particularise l’élément a n’est pas la valeur de l’attribut class mais l’état de l’élément.)
• En CSS1 on a les pseudo-classes a:link (lien non visité),
a:visited (lien visité) et a:active (lien actif).
a:link
a:visited
Si on veut utiliser des pseudo-classes, il est important de les mentionner dans cet ordre : link, visited et active.
